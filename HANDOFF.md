# 项目接手说明

更新日期：2026-06-19
当前稳定基线：`main`；即时学习模式 checkpoint 为 `759c5b6`

## 1. 当前状态

这是一个可离线运行的纯前端单片机复习刷题网页，无后端、无数据库、无 npm 依赖。页面可直接打开 `index.html`，也可通过 GitHub Pages 在 iPhone 上安装到主屏幕。

当前题库共 200 题：

- 单选题 82 道
- 多选题 43 道
- 判断题 75 道
- 已核验解析 89 道
- 待人工复核解析 111 道
- 解析置信度：高 33 道、中 56 道、低 111 道

当前版本已经过题库重新生成验证，`node parser.js` 输出上述题数和解析状态统计。

## 2. 已完成能力

### 做题流程

- 专注做题页与“设置与数据”页分离。
- 支持全部、未做、历史错题、错误次数阈值、错误率、最近一次做错、收藏、未掌握等范围。
- 支持 10 / 20 / 50 / 全部题量，顺序或随机出题。
- 支持立即判题和批量提交。
- 完整支持单选、多选、判断题。
- 支持上一题、下一题、题目导航和键盘快捷键。
- 支持在提交前将拿不准的题目标记为“不确定”，立即展示答案、解析和知识点，并在本轮单列为不计对错的“待复习”题。
- 本轮结束后显示统计、错题复查、重练错题和再练本轮。

### 状态持久化

- 学习记录、练习偏好、主题和当前轮次均保存到 `localStorage`。
- 刷新或重新打开同一浏览器地址可恢复当前轮次和未提交答案。
- 支持导入/导出学习数据、导出错题列表和二次确认清空记录。
- 学习记录只存在当前浏览器，不会在电脑和手机间自动同步。
- GitHub Pages 使用固定 HTTPS 地址；PWA 首次在线加载后可离线启动，更新缓存不会清除学习记录。

### 界面

- 提供浅色实验室和 GitHub Dark 两套主题。
- 电脑端为主要优化目标，同时已适配平板和手机窄屏。
- 题目导航默认收起，减少做题干扰。

### 题目解析

- `explanations.json` 是解析 sidecar，`parser.js` 会将其与 `questions.txt` 合并生成 `questions.js`。
- 解析 sidecar schema v2 要求 200 题都包含非空 `knowledgePoint`，生成后页面会在解析区展示知识点。
- 页面只在判题后显示解析。
- `verified` 会显示核验标识和来源链接；`needs_review` 明确标记“待人工复核”，不冒充可靠结论。
- 解析器会检查 200 题覆盖、答案一致、reviewQueue 对应关系、HTTPS 来源和可信域名。
- PJRC 上的 Philips 80C51 原厂手册镜像按中等置信度处理，不视为官方站点直链。

## 3. 数据与关键约束

题库数据链路：

```text
questions.txt + explanations.json
              ↓ node parser.js
          questions.js
              ↓
           index.html
```

不要手动编辑 `questions.js`。修改题目应改 `questions.txt`；修改解析应改 `explanations.json`；之后必须运行 `node parser.js`。

`localStorage` key：

- `mcu_exam_practice_records_v1`：逐题学习记录
- `mcu_exam_practice_theme_v1`：主题
- `mcu_exam_practice_session_v1`：当前练习轮次
- `mcu_exam_practice_preferences_v1`：练习设置

必须保持旧数据兼容，不能无迁移地修改这些 key 或现有记录字段。

静态与题库检查：

```powershell

```

修改做题逻辑后至少验证：

- 单选、多选、判断各一题。
- 立即判题和批量提交各一轮。
- 刷新后恢复当前轮次与草稿。
- 错题记录、收藏、掌握、不确定状态和筛选正确；查看答案不增加做题次数或对错次数。
- 导入/导出学习记录。

修改 UI 后至少验证桌面与手机窄屏、两套主题、控制台无错误。优先使用 in-app Browser；若其截图或下载能力失败，再记录原因并使用外部浏览器回退。

## 6. 已知风险与后续优先级

### P0：解析可信度闭环

111 道 `needs_review` 仍需逐题人工或可靠资料复核。这些内容当前可作参考，但不能视为最终正确。任何升级为 `verified` 的条目都应满足：答案一致、解释与题意匹配、来源可访问、引用章节能直接支持结论。

### P1：解析复核工作流

下一轮高价值改进建议：

- 增加“已核验解析题”和“待复核解析题”练习范围。
- 增加“导出待复核题目”JSON，包含题干、选项、答案、当前解析、置信度、复核原因和建议，便于交给网页端 GPT 整理后再审。
- 在设置页当前题历史中显示解析状态，但不要增加做题页干扰。

### P2：跨设备同步

当前电脑和手机不会自动同步。只有用户明确需要多设备同步时，才考虑云端账号/数据库；在此之前继续用 JSON 导入导出最符合本地、离线、低复杂度目标。
