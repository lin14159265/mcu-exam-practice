# 单片机期末复习刷题系统

这是一个纯前端刷题网页，用于练习 `questions.txt` 中的单片机期末复习题。页面不需要后端或数据库，联网打开一次后可离线使用，学习记录保存在浏览器 `localStorage` 中。

开发者或新对话接手工程时，请先阅读 `AGENTS.md` 和 `HANDOFF.md`；后者记录当前完成度、验证基线、已知风险与后续优先级。

## 文件说明

- `index.html`：网页入口，双击即可用浏览器打开。
- `style.css`：页面样式和响应式布局。
- `app.js`：刷题、判题、学习记录、导入导出逻辑。
- `questions.txt`：原始题库文本。
- `explanations.json`：题目解析、核验状态和来源信息。
- `parser.js`：题库解析脚本。
- `questions.js`：由 `parser.js` 自动生成的结构化题库数据。
- `manifest.webmanifest` / `service-worker.js` / `icon*.png`：主屏幕安装信息、离线缓存和应用图标。

## 生成题库数据

首次使用或修改 `questions.txt` 后，在当前目录运行：

```bash
node parser.js
```

成功后会合并 `questions.txt` 与 `explanations.json`，生成或覆盖 `questions.js`，并输出题目总数、题型和解析状态统计。当前解析器支持常见格式：

- `1. 题干`
- `1、题干`
- `第1题 题干`
- `A. 选项`
- `A、选项`
- `答案：A`
- `正确答案：A`
- 判断题答案 `对` / `错`
- 多选题答案如 `ABD`

生成前会自动检查：

- 解析必须覆盖全部题号，不能缺题或多出题号。
- 解析中的答案必须与题库答案完全一致。
- `needs_review` 必须与 `reviewQueue` 一一对应。
- `verified` 必须有 HTTPS 来源；高置信度只接受芯片厂商官方域名。
- Philips 原厂 80C51 手册的 PJRC 镜像仅按中等置信度接入。

当前数据中有 89 道已核验解析、111 道待人工复核解析。待复核内容会在页面明确标识，不会显示成官方结论。

## 打开网页

确认 `questions.js` 已生成后，直接双击 `index.html`，或在浏览器中打开该文件即可。

如果你想用 `http://127.0.0.1:8765/index.html` 访问，或者 in-app browser 提示打不开，请双击：

```text
start-local.bat
```

脚本会在当前目录启动本地静态服务并自动打开网页。使用期间不要关闭脚本窗口；需要停止服务时，在脚本窗口按 `Ctrl+C`。

如果提示 `Port 8765 is already in use`，说明 8765 端口已经被其他程序占用。若浏览器已经能打开页面，可以继续使用；否则关闭占用该端口的程序，或编辑 `start-local.bat` 里的 `PORT` 值后重试。

## 页面视图和主题

页面顶部有三个主要入口：

- `做题`：专注刷题页面，只保留题目、选项、判题结果和必要操作。
- `设置与数据`：练习范围、本轮题数、出题顺序、判题方式、导入导出、题目历史和本轮统计。
- `GitHub Dark / 浅色实验室`：切换配色主题。

主题选择会保存到浏览器 `localStorage` 的 `mcu_exam_practice_theme_v1`。学习记录仍保存到 `mcu_exam_practice_records_v1`，切换主题不会影响做题数据。

练习范围、题数、顺序和判题方式会保存到 `mcu_exam_practice_preferences_v1`。下次打开页面可直接按上次设置开始，不需要重复配置。

当前轮次会单独保存到 `mcu_exam_practice_session_v1`。刷新或重新打开同一浏览器地址时，会自动恢复题目顺序、当前位置和已选答案；点击“重置当前练习”会清除该轮次，但不会删除历史学习记录。

题目解析只在判题后显示。已核验解析会附带文档发布方、编号、章节及来源链接；来源链接需要联网打开，但离线刷题和解析正文不受影响。

## 学习记录

学习记录自动保存到当前浏览器的 `localStorage`。同一个浏览器、同一个站点地址下刷新页面或更新离线缓存不会丢失记录。

页面提供：

- 导出学习数据：保存当前全部学习记录、练习偏好和主题为 JSON。
- 导入学习数据：选择之前导出的 JSON 文件恢复；旧版仅含学习记录的 JSON 仍可导入。
- 清空学习记录：会弹出二次确认，清空后不可恢复，除非你已提前导出记录。
- 导出错题列表：导出所有做错过的题目及错误次数、错误率、最近一次错误答案。

导入或清空学习记录会结束当前轮次，避免恢复后的历史记录与页面上旧的答题状态不一致。

## 电脑端快捷操作

快捷键仅在做题页生效，设置页、下拉框和题目导航打开时不会触发：

- `A` / `B` / `C` / `D` 或 `1` / `2` / `3` / `4`：选择对应选项。
- `Enter`：提交多选题；判题后进入下一题；批量模式下进入下一题。
- `←` / `→`：上一题 / 下一题。
- `Shift + Enter`：上一题。
- `Ctrl + Enter`：批量模式提交本轮答案。
- `G`：打开题目导航。

## 解析失败时如何检查

如果运行 `node parser.js` 后出现解析错误，请按控制台提示定位对应题块，重点检查：

- 题号是否独占题块开头，例如 `83.（多选题）...`。
- 每个选项是否以 `A.`、`A、`、`B.` 等格式开头。
- 是否至少有两个选项。
- 是否有 `答案：` 或 `正确答案：` 行。
- 答案字母是否能对应到已有选项；判断题的 `对` / `错` 是否在选项中。

修正 `questions.txt` 后重新运行 `node parser.js`。

## 当前功能

- 全部、未做、错题、高错误次数、高错误率、最近一次做错、收藏、未掌握筛选。
- 顺序和随机出题。
- 10 题、20 题、50 题、全部组卷。
- 立即判题和批量提交。
- 上一题/下一题导航和本轮完成进度；批量模式选择答案后自动保存。
- 刷新页面后自动恢复当前轮次和未提交草稿。
- 本轮完成页展示题数、答对、答错和正确率，并支持点开错题复查。
- 可一键用立即判题模式重练本轮错题，或按原模式再练本轮题目。
- 题目导航默认收起，可快速定位当前题、未答题、已答题和本轮错题；随机练习同时显示题库原题号。
- 单选题、多选题、判断题。
- 自动记录做题次数、正确次数、错误次数、错误率、连续答对、掌握状态、不确定状态和收藏状态。
- 可在独立提交前点击“不确定 / 查看答案”，立即学习标准答案、正确选项、解析和知识点；该题进入未掌握范围，但本次不计答对、答错或做题次数。
- 本轮结果会单列“待复习”题，正确率只统计独立作答题；重新独立连续答对 3 次后自动掌握。
- 连续答对 3 次自动标记为已掌握，可手动取消；手动标记掌握会同步清除“不确定”状态。
